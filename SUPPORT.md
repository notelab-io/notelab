# Support

Use the right channel so the project can respond effectively.

## Questions and Ideas

Use GitHub Discussions if they are enabled. Otherwise, open a GitHub issue and choose the feature request or question path.

## Bugs

Open a bug report with reproduction steps, environment details, expected behavior, actual behavior, and logs or screenshots when relevant.

## Security

Do not open public issues for vulnerabilities. See `SECURITY.md` and report privately through GitHub private vulnerability reporting or `hey@sreeragh.me`.

## Self-Hosting

Start with:

- `README.md`
- `https://docs.notelab.io/self-hosting`

When asking for help, include your deployment method, relevant environment variable names without secret values, container status, and logs with secrets redacted.
